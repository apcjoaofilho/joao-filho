# Vibe coding sem fantasia

Vibe coding é útil.

Mas só quando entra no lugar certo do processo.

Se vira licença para não pensar, o resultado é previsível: código frágil, deploy quebrado, retrabalho e uma falsa sensação de velocidade.

## O que vibe coding acelera de verdade

- sair da página em branco
- testar uma ideia rápido
- explorar alternativas
- estruturar um rascunho inicial
- resumir documentação e reduzir atrito cognitivo

Quando a tarefa ainda está em formação, esse tipo de ajuda funciona muito bem.

## O que vibe coding não substitui

- entendimento do problema
- revisão de risco
- teste de integração
- validação do comportamento real
- decisão de arquitetura

O ponto central é este: **assistência acelera execução, mas não substitui julgamento**.

## O erro mais comum

O erro mais comum é pedir solução antes de formular direito o problema.

Nesse cenário, o modelo ou agente até entrega algo.
Só que responde à formulação errada.

O efeito parece produtivo por alguns minutos, mas o custo volta depois:

- nomes ruins
- dependências desnecessárias
- fluxo quebrado
- edge cases ignorados
- explicações bonitas para decisões ruins

## O meu critério atual

Eu trato vibe coding como um multiplicador de velocidade em quatro etapas:

1. esclarecer o problema
2. gerar um primeiro caminho
3. revisar criticamente
4. validar no mundo real

Se a etapa 3 ou 4 some, não é aceleração.
É só terceirização do erro.

## O que realmente melhora o resultado

O que melhora o resultado não é “usar IA para codar”.

É usar IA dentro de um sistema:

- contexto bom
- escopo claro
- diffs pequenos
- validação frequente
- deploy observável

Quanto mais crítico o contexto, menos espaço existe para improviso.

## Um jeito mais maduro de pensar

Eu prefiro pensar assim:

> vibe coding é ótimo para exploração;
> engenharia continua sendo essencial para decisão, validação e manutenção.

Esse enquadramento evita dois extremos:

- o tecnofetichismo de achar que tudo agora é automático
- o purismo de fingir que ferramentas novas não mudam nada
